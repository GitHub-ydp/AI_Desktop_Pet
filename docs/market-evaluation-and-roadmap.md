## 三、致命风险（必须在公开发布前解决）

### 风险1：API Key 暴露在客户端

```
当前链路：客户端 → dashscope.aliyuncs.com（Key 硬编码在 builtin-api.js）
风险：反编译 → 提取 Key → 盗用 → 费用失控
必须改为：客户端 → 你的后端网关（鉴权+限流）→ dashscope
```

**严重程度：P0，发布前必须解决。**

### 风险2：MIT 开源协议

```
当前：MIT License，任何人可 fork + 替换 Key + 上线竞品
建议：评估改为 BSL（Business Source License）或 SSPL
     保留源码可见，但限制商业复制
```

### 风险3：无成本控制

```
每次 AI 对话 = API 调用 = 花钱
没有账号 + 没有限流 = 无限烧钱
保守估算：1000 日活 × 20条/天 × ¥0.01/条 = ¥200/天 = ¥6000/月
1万日活 = ¥6万/月，无收入对冲
```

## 四、阶段性目标

### 阶段一：商业化基础（P0，4-6 周）

> **目标：让项目能安全上线、能控制成本、能开始收费**

#### 1.1 后端 API 网关（第 1-2 周）

```
技术方案：
- 轻量 Node.js 服务（Express/Fastify）
- 部署：阿里云函数计算 或 轻量服务器
- 功能：
  ├── 接收客户端请求（带 JWT token）
  ├── 校验用户身份 + 检查配额
  ├── 转发到 Qwen API（Key 仅存服务端）
  └── 记录用量（用户ID、tokens、时间戳）
- 客户端改造：
  ├── builtin-api.js 端点改为指向自己的后端
  └── 请求附带 JWT token
```

**验收标准：** API Key 不再出现在客户端代码中

#### 1.2 用户账号系统（第 2-3 周）

```
最小 MVP：
- 注册/登录：手机号 + 验证码（阿里云短信 ¥0.045/条）
- 存储：云数据库（MySQL/PostgreSQL）
- 鉴权：JWT token，本地持久化
- 客户端：
  ├── 首次打开 → 登录/注册页面
  ├── 登录后 token 存入 localStorage
  └── 所有 API 请求附带 token

数据库表：
- users（id, phone, created_at, subscription_tier, subscription_expires_at）
- usage_logs（user_id, date, message_count, token_count）
```

#### 1.3 订阅套餐 + 支付（第 3-5 周）

```
套餐设计：
┌──────────────────────────────────────────────────────────┐
│  免费版（¥0）                                             │
│  - 每天 30 次对话                                         │
│  - 基础皮肤（猫咪）                                       │
│  - 7 天记忆保留                                           │
│  - 无 Agent 技能                                          │
├──────────────────────────────────────────────────────────┤
│  标准版（¥9.9/月）                                        │
│  - 每天 200 次对话                                        │
│  - 全部皮肤                                               │
│  - 永久记忆                                               │
│  - Agent 技能（文件/搜索/提醒）                            │
├──────────────────────────────────────────────────────────┤
│  Pro 版（¥29.9/月）                                       │
│  - 无限对话                                               │
│  - 全部功能                                               │
│  - 优先响应                                               │
│  - 云端记忆同步（未来）                                    │
└──────────────────────────────────────────────────────────┘

支付接入：微信支付 + 支付宝（使用聚合支付如 Payjs/虎皮椒，接入成本最低）
```

#### 1.4 协议变更评估（第 1 周）

评估从 MIT 改为 BSL 或其他限制商业复制的协议。

**阶段一完成标志：** 可以安全公开发布，有基本的成本控制和收入入口。

### 阶段二：分发与增长（6-10 周）

> **目标：获取首批 1000 用户**

#### 2.1 官网 / 落地页（1 周）

```
必备内容：
- 30秒 GIF / 视频展示核心体验
- 功能亮点：记忆、Agent、截图、提醒
- 下载按钮（Windows/Mac/Linux）
- 订阅入口
- 技术栈：静态页面即可（Vercel/Netlify 免费部署）
```

#### 2.2 内容营销（持续）

```
核心渠道：B站 + 抖音 + 小红书

内容方向：
- "AI 桌宠帮我整理了一周的工作"    → Agent 能力展示
- "我教会了桌宠认识我的猫"          → 记忆系统展示
- "这个 AI 发现我最近情绪不好"      → 情感陪伴展示
- "桌宠 vs ChatGPT 有什么不一样"    → 差异化对比

关键指标：每条视频的下载转化率
```

#### 2.3 皮肤内容扩充

```
优先级：
① 为 狗狗🐶、兔子🐰 补充 Lottie 动画（至少 idle/happy/sleeping）
② 节日限定皮肤（免费引流 or 付费）
③ 准备皮肤商店 UI 框架

目标：从 1 个有动画的宠物 → 至少 3 个
```

#### 2.4 社交裂变机制

```
- 邀请好友注册 → 双方各获 3 天标准版体验
- 对话金句 → 一键生成分享卡片（带品牌水印）
- 周报总结 → 可分享的「我和桌宠的一周」卡片
```

**阶段二完成标志：** DAU 破 1000，付费转化率 > 3%。

### 阶段三：产品深化（10 周以后）

> **目标：提升留存和 ARPU**

| 方向 | 价值 | 难度 | 说明 |
|------|:----:|:----:|------|
| 宠物养成体系 | 高 | 中 | 亲密度+等级+解锁内容，提升日活 |
| 记忆可视化 | 高 | 低 | 展示 user_profile，让用户感知记忆价值 |
| 云端记忆同步 | 高 | 中 | 跨设备体验，增强锁定 |
| 移动端（小程序起步）| 极高 | 高 | 用户规模×10，但开发量大 |
| 皮肤商城 + UGC | 中 | 中 | 创作者分成，生态壁垒 |
| 语音交互 | 中 | 中 | TTS+ASR，情感差异化 |
| 企业版 | 中 | 高 | B端定价高，但需求验证 |

## 五、关键商业指标体系

```
核心指标：
- DAU / MAU              （日活 / 月活）
- 付费转化率             （目标 5-10%）
- ARPU                   （平均每用户月收入）
- API 成本 / 用户 / 月    （控制在 ¥2 以内）
- 7日留存 / 30日留存      （目标 40% / 20%）
- LTV / CAC              （用户终身价值 / 获客成本）

盈亏平衡估算：
- API 成本：¥0.01/条 × 50条/天 = ¥0.5/用户/天 = ¥15/用户/月
- 标准版收入：¥9.9/月
- 需要 Pro 版和增值服务拉高 ARPU
- 目标 ARPU ≥ ¥20/月 → 毛利率 25%+
```

## 六、执行优先级速查

```
立刻做（第 1-6 周）:
  ① 后端 API 网关 — 去掉客户端 Key 暴露（2周）
  ② 用户账号系统 — 注册/登录/JWT（2周）
  ③ 订阅套餐 + 支付接入（2周）
  ④ 协议评估（MIT → BSL？）

接着做（第 6-10 周）:
  ⑤ 落地页 / 官网
  ⑥ 皮肤内容扩充（至少3个有动画）
  ⑦ 内容营销启动（B站/抖音）
  ⑧ 社交裂变（邀请/分享卡片）

长期做（10 周+）:
  ⑨ 宠物养成 + 记忆可视化
  ⑩ 移动端（微信小程序起步）
  ⑪ 皮肤商城 + 创作者分成
  ⑫ 语音交互
```

## 七、一句话总结

> **技术底座已经就位，内置 API 解决了最大的用户门槛。现在最紧迫的不是加功能，而是建后端——没有后端网关和账号系统就上线，等于免费给用户烧钱，项目活不过 3 个月。**
