# AI Desktop Pet 产品收敛与执行计划

> 日期：2026-03-12
> 目标：把当前“功能广而散”的项目，收敛成一个可持续迭代、可验证商业价值的产品
> 原则：先聚焦，再完善；先验证留存，再扩张功能；先让用户用起来，再考虑做大

---

## 1. 当前判断

### 1.1 项目现状

当前项目已经具备以下基础能力：

- 桌面宠物形态与陪伴式交互
- 多人格对话
- 长期记忆与事实提取
- 提醒、任务、天气等日常功能
- 截图、OCR、翻译、分析
- 文件和系统操作工作流
- 技能系统与 MCP 扩展能力

这些能力说明项目的技术基础足够强，但产品问题也很明确：

- 功能过多，首屏价值不清晰
- 用户第一次打开时，不知道核心价值是什么
- API Key 门槛太高，严重影响体验
- 内容层不够强，宠物和陪伴感没有被充分感知
- 付费路径不存在，商业闭环未建立

### 1.2 产品收敛结论

未来 3 个月不再追求“做更多能力”，而是集中把产品收敛为：

**一个会陪你聊天、记住你、并能帮你处理电脑日常任务的 AI 桌面搭子。**

这个定位下，优先保留的价值主线只有 3 条：

1. 陪伴与人格感
2. 长期记忆与关系成长
3. 桌面高频小任务协助

不属于这 3 条主线的功能，未来 12 周一律降级为次优先级。

## 2. 目标用户

### 2.1 第一阶段目标用户

先只服务一类核心用户，不同时做泛用户市场。

建议首批目标用户：

- 长时间使用电脑的学生
- 长时间使用电脑的办公/内容工作者
- 愿意接受桌宠外形，但更看重“陪伴 + 小工具效率”的用户

### 2.2 用户核心需求

这类用户最真实的需求不是“有个会聊天的宠物”，而是：

- 在电脑前有一个持续陪伴感的存在
- 能记住我，而不是每次重新认识我
- 能帮我做一点轻量但高频的事
- 不折腾配置，打开就能用

## 3. 产品策略

### 3.1 暂定产品定位

一句话定位：

**AI Desktop Pet 是一个会陪你聊天、记住你、还能帮你处理电脑日常任务的 AI 桌面搭子。**

### 3.2 产品主打场景

只打以下 4 个高频场景：

1. 聊天陪伴
2. 日常提醒与任务
3. 截图即问、截图即翻译、截图即理解
4. 轻量桌面操作

### 3.3 暂时不重点推进的方向

以下方向先不作为主线投入：

- 大而全的 Agent 自动化
- 复杂工作流编排
- 皮肤商城
- UGC 创作者生态
- 多端同步
- 语音系统
- 复杂文件管理平台化

这些方向不是不做，而是必须排在“产品收敛和留存验证”之后。

## 4. 接下来 12 周执行路线

## 阶段 A：产品收敛与首屏体验

时间：第 1-3 周

目标：

- 用户第一次打开时，能在 3 分钟内理解产品价值
- 用户不需要理解全部功能，只要先感受到 1 个明确亮点

本阶段必须完成：

1. 明确首页和启动后的核心叙事
2. 调整首次使用引导
3. 压缩用户一开始能看到的功能面
4. 降低 API Key 使用门槛

具体任务：

1. 统一产品文案
- 所有首页、README、设置页、站点文案统一成一个定位
- 删掉过于技术化、堆功能式的表达

2. 改造首次启动流程
- 第一次启动只展示 3 个核心能力
- 给出可点击的示例操作
- 用户第一次就能试一次聊天、一次截图提问、一次提醒

3. 调整信息架构
- 先隐藏高级功能入口
- MCP、技能、复杂设置暂时弱化
- 主界面只突出“聊天 / 截图 / 提醒 / 记忆”

4. 处理 API Key 门槛
- 方案优先级 A：接入官方托管中转服务
- 方案优先级 B：提供“试用模型”或公共免费额度
- 方案优先级 C：本地模式和自带 Key 模式并存

验收标准：

- 新用户首次打开后 1 分钟内可以完成一次核心交互
- 首次启动不再出现“必须先配 API Key 才能体验核心功能”的卡点
- 主界面功能入口数量明显收敛

---

## 阶段 B：强化留存闭环

时间：第 4-6 周

目标：

- 用户有理由第二天再打开
- 用户能感知到“它记得我”和“它在成长”

本阶段必须完成：

1. 记忆可视化
2. 亲密度成长闭环
3. 每日轻仪式感

具体任务：

1. 完善“我的记忆”页面
- 展示宠物记住的用户信息
- 展示最近记住的事实和偏好
- 支持删除错误记忆
- 支持标记“重要记忆”

2. 升级亲密度系统
- 现在已有积分基础，补齐等级反馈
- 每次升级要有明显视觉反馈和文案反馈
- 等级解锁具体奖励，不允许只加数值

3. 做每日使用理由
- 每日首次打开问候
- 晚间回顾
- 本周聊天摘要
- 任务完成时的正反馈

4. 收敛提醒和任务体验
- 提醒一定要简单、稳定、可信
- 任务创建流程缩短
- 完成任务后要反馈到成长系统

验收标准：

- 用户能明确看到系统记住了什么
- 用户能感知到“使用一次就积累一次关系”
- 有至少 2 个每天值得打开的理由

---

## 阶段 C：打造一个可付费 MVP

时间：第 7-9 周

目标：

- 做出第一个可以测试付费意愿的版本
- 不追求完整商业系统，先验证“是否有人愿意为这个产品买单”

本阶段必须完成：

1. 免费版 / 付费版分层
2. 明确权益边界
3. 建立最小转化路径

建议版本分层：

### 免费版

- 基础聊天
- 单宠物形象
- 有限记忆
- 每日有限截图分析次数
- 基础提醒

### Pro 版

- 完整长期记忆
- 更多宠物形象
- 完整截图/OCR/翻译能力
- 任务与提醒增强
- 更强模型或更高额度

具体任务：

1. 定义付费权益文档
- 哪些功能属于免费
- 哪些功能属于付费
- 为什么这些功能值得付费

2. 增加产品内提示
- 当用户使用到关键能力时自然提示价值
- 不做强打断式弹窗

3. 准备最小支付闭环
- 先支持最简单的订阅或激活码方案
- 不要在这一阶段做复杂账号体系

验收标准：

- 有清楚的免费/付费边界
- 至少能对外发布一个可试运营的 Pro 方案
- 能统计多少用户点击、试用、付费

## 阶段 D：为增长做准备

时间：第 10-12 周

目标：

- 在不破坏产品主线的前提下，增加自传播能力

本阶段必须完成：

1. 对话或回顾卡片分享
2. 角色内容增强
3. 对外传播页面和下载流程优化

具体任务：

1. 增加分享内容
- 每周摘要卡
- 宠物金句卡
- 成长里程碑卡

2. 补齐内容层
- 至少做 3 个完整可用角色
- 每个角色有基础动作和差异化文案

3. 优化官网和下载页
- 首页只强调 3 个卖点
- 下载流程清晰
- 展示真实使用场景，而不是堆技术名词

验收标准：

- 产品具备最基本的自然传播素材
- 用户下载后路径更短
- 产品不再像一个“开发者作品集”

## 5. 每周执行机制

为了避免再次回到“功能越做越多，完成度越来越差”的状态，执行机制必须改变。

### 5.1 每周只做一个主题

每周只能有一个核心主题，例如：

- 首次体验周
- 记忆可视化周
- 提醒闭环周
- 付费权益周

禁止一周内同时推进 4-5 条大线。

### 5.2 任务分级

所有任务只能分成三类：

#### P0

必须做，不做会直接影响产品主线

#### P1

重要，但可以延后一周

#### P2

想做但不影响当前阶段目标

每周开始前必须明确本周 P0 不超过 3 项。

### 5.3 完成定义

以后“完成”不能只代表代码写了，还必须同时满足：

1. 功能可用
2. 路径可发现
3. 文案可理解
4. 至少手工验证过一次主流程
