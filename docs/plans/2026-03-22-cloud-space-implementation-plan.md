### Plus 用户（高级版）

**谁：** 25-35 岁，月薪 8K+，有轻度技术需求但不想折腾运维

**场景：**
- "把文件发给朋友但不想用百度网盘"→ 上传返回链接
- "做个生日邀请页发给朋友"→ 部署到子域名
- "帮我跑个 Python 数据分析"→ 云端执行返回结果
- "博客需要图床"→ 稳定可控的公网文件托管

**决策关键词：** 零配置、立刻可用、有面子（自己的域名）

### Pro 用户（专属服务器）

**谁：** 25-40 岁，已有 VPS，懂 Linux，但运维是负担不是乐趣

**场景：**
- "帮我重启 nginx"→ 一句话完成
- "部署这个项目到服务器"→ git pull + build + restart 自动完成
- "每天检查服务是否正常"→ 定时监控 + 异常通知
- "看看磁盘还剩多少"→ 即时报告

**决策关键词：** 安全、可控、比手动 SSH 快 10 倍

### 5.2 Phase 1 核心技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 容器管理 | **Dockerode**（Node.js Docker API 客户端） | 纯 JS，与后端 Express 一体 |
| 反向代理 | **Caddy v2** | 自动 HTTPS + Admin API 动态配置，不用 reload |
| 文件传输 | **docker cp**（via Dockerode putArchive） | 无需容器开 SSH 端口 |
| 脚本沙箱 | **docker exec** | 容器本身就是沙箱 |
| 定时任务 | 容器内 **cron** | 隔离，不影响宿主机 |
| 基础镜像 | **debian-slim + python3 + cron** | 轻量且够用 |

### 5.3 Phase 2 核心技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| SSH 客户端 | **ssh2**（npm） | 纯 JS，无需 native rebuild |
| 密钥存储 | **AES-256-GCM**（crypto 内置） | 机器 ID 派生密钥，绑定本机 |
| 文件传输 | **SFTP**（ssh2 内置） | 标准协议，可靠 |
| 命令审查 | **正则三级分类** | blocked/confirm/audit |
| 审计日志 | **SQLite**（复用现有 pet-memory.db） | 不引入新依赖 |

### 5.4 容器规格

```
每用户容器配置：
  CPU:    1 核（共享，实际并发约 0.15 核/用户）
  内存:   512 MB（硬限制，禁用 swap）
  磁盘:   5 GB（overlay2 size 限制）
  进程:   100 个上限（防 fork bomb）
  网络:   隔离 bridge network
  root:   只读根文件系统 + /tmp 临时写入
  用户:   sandbox（uid=1000，非 root）
```

### 5.5 安全架构（5 层）

```
┌───────────────────────────────────────────────────────┐
│ L1 网络层    HTTPS 强制 / Rate Limit 100req/min/user  │
│              容器无公网 SSH / Docker 网络隔离           │
├───────────────────────────────────────────────────────┤
│ L2 认证层    JWT 24h 过期 / 订阅级别中间件校验         │
│              SSH 密钥 AES-256-GCM 加密（机器绑定）     │
├───────────────────────────────────────────────────────┤
│ L3 沙箱层    read-only rootfs / cap-drop ALL           │
│              pids-limit 100 / no-new-privileges        │
├───────────────────────────────────────────────────────┤
│ L4 审查层    硬阻断（rm -rf / / fork bomb / dd）       │
│              需确认（sudo rm / DROP TABLE / kill -9）   │
│              审计记录（sudo / apt / systemctl）         │
├───────────────────────────────────────────────────────┤
│ L5 审计层    所有操作写 SQLite 日志 / 文件覆盖前备份   │
│              90 天保留 / 用户可查看可回滚               │
└───────────────────────────────────────────────────────┘
```

## 七、后端 API 接口

### 云空间 API（全部需要 JWT + 订阅校验）

```
# 容器
POST   /api/v1/cloud/containers/init      初始化用户容器
GET    /api/v1/cloud/containers/status     容器状态
POST   /api/v1/cloud/containers/wake       唤醒暂停容器
GET    /api/v1/cloud/containers/stats      资源用量

# 文件
POST   /api/v1/cloud/files/upload          上传（multipart, ≤50MB/次）
GET    /api/v1/cloud/files/list            列出文件
GET    /api/v1/cloud/files/download        下载文件
DELETE /api/v1/cloud/files                 删除文件
POST   /api/v1/cloud/files/mkdir           创建目录

# 脚本
POST   /api/v1/cloud/scripts/run           执行 Python 脚本
GET    /api/v1/cloud/scripts/history        执行历史

# 定时任务
POST   /api/v1/cloud/cron                  创建
GET    /api/v1/cloud/cron                  列出
DELETE /api/v1/cloud/cron/:id              删除
POST   /api/v1/cloud/cron/:id/run          手动触发

# Web 发布
GET    /api/v1/cloud/web/url               获取子域名 URL
POST   /api/v1/cloud/web/deploy            快捷部署
POST   /api/v1/cloud/web/toggle            开关公网访问
```
