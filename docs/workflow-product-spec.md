# 工作流执行能力 - 产品规范文档

> 版本：v1.0 MVP | 日期：2026-02-22

## 1. 功能概述

使 AI 桌面宠物能通过**自然语言对话**驱动桌面文件操作，后端由 Python 工具脚本执行。用户无需记忆命令，只需像和朋友说话一样描述需求，宠物即可理解并代为操作。

---

## 2. 功能范围（MVP）

### P0 - 核心必需（首版必须交付）

| 工具 | 描述 | 风险等级 |
|------|------|----------|
| **文件搜索** | 按名称/类型/修改时间搜索文件 | 低 |
| **打开文件** | 使用系统默认程序打开文件 | 低 |
| **创建文件夹** | 在指定路径创建文件夹 | 低 |
| **文件信息查看** | 查看文件大小、修改时间等元数据 | 低 |

### P1 - 重要增强（首版尽量交付）

| 工具 | 描述 | 风险等级 |
|------|------|----------|
| **文件复制** | 复制文件到指定目录 | 中 |
| **文件移动/重命名** | 移动或重命名文件 | 中 |
| **批量整理** | 按规则分类整理文件（如按后缀归类） | 中 |
| **文件内容预览** | 读取文本文件前 N 行 | 低 |

### P2 - 进阶功能（后续迭代）

| 工具 | 描述 | 风险等级 |
|------|------|----------|
| **文件删除** | 删除文件（移至回收站） | 高 |
| **文件压缩/解压** | ZIP 压缩与解压 | 中 |
| **重复文件检测** | 扫描重复文件并建议清理 | 中 |
| **磁盘空间分析** | 分析文件夹占用空间 | 低 |

## 4. UX 交互规范

### 4.1 执行前 - "即将执行"确认

**展示方式：** 在聊天窗口中以**操作卡片**形式展示，区别于普通文字消息。

```
┌─────────────────────────────────┐
│  [工具图标] 即将执行操作          │
│                                 │
│  操作：复制文件                   │
│  源文件：C:\桌面\简历.pdf        │
│  目标：D:\备份\                  │
│                                 │
│  [ 执行 ]    [ 取消 ]           │
└─────────────────────────────────┘
```

**设计要点：**
- 卡片使用 `var(--bg-card)` 背景 + `var(--border)` 边框，与聊天主题一致
- 操作类型使用图标区分：搜索(🔍)、打开(📂)、创建(📁)、复制(📋)、移动(➡️)、删除(🗑️)
- 低风险操作（搜索、打开、查看信息）可跳过确认，直接执行
- 中风险操作（复制、移动、重命名）需用户点击"执行"
- 高风险操作（删除）需**二次确认**（见 4.5）

### 4.2 执行中 - 宠物状态

**动画状态：** 复用 `thinking` 状态（思考气泡动画）

**执行时长提示：**
- < 3 秒：仅显示宠物思考动画 + 一句执行台词
- 3~10 秒：显示进度指示（"正在处理中..."）
- \> 10 秒：显示进度百分比（如批量操作）

### 4.3 执行成功 - 宠物反应

**动画状态：** 切换到 `happy` 状态 + 闪光粒子效果

**聊天反馈：** 在聊天窗口显示结果摘要卡片：

```
┌─────────────────────────────────┐
│  ✅ 操作完成                     │
│                                 │
│  已复制 简历.pdf 到 D:\备份\     │
│  文件大小：2.3 MB               │
│                                 │
│  [ 打开文件 ] [ 打开所在文件夹 ] │
└─────────────────────────────────┘
```

### 4.4 执行失败 - 友好错误展示

**动画状态：** 切换到 `sad` 状态 + 泪滴粒子

**错误展示：** 用宠物的口吻描述问题，而非技术错误码。

```
┌─────────────────────────────────┐
│  ⚠️ 操作遇到问题                 │
│                                 │
│  [宠物台词：根据性格不同]         │
│  原因：文件被其他程序占用了       │
│                                 │
│  [ 重试 ] [ 算了 ]              │
└─────────────────────────────────┘
```

**错误翻译对照：**

| 技术错误 | 友好表述 |
|----------|----------|
| `ENOENT` | 找不到这个文件呢，路径对吗？ |
| `EACCES` | 没有权限操作这个文件哦 |
| `EBUSY` | 文件被其他程序占用了 |
| `ENOSPC` | 磁盘空间不够了 |
| `EPERM` | 需要管理员权限才能操作 |
| `EEXIST` | 目标位置已经有同名文件了 |
| 超时 | 操作花了太长时间，可能文件太大了 |

### 4.5 危险操作确认流程（删除文件）

**二次确认弹窗** 使用内联对话框（与设置窗口中的 confirm 替换方案一致）：

```
第一步（聊天中）：
宠物："你确定要删除 临时文件.txt 吗？"
[ 确认删除 ]  [ 取消 ]

第二步（点击确认后，内联弹窗）：
┌──────────────────────────────────┐
│  ⚠️ 再次确认                      │
│                                  │
│  即将删除：临时文件.txt           │
│  此操作会将文件移至回收站          │
│                                  │
│  输入文件名以确认：[___________]  │
│                                  │
│  [ 确认删除 ]  [ 我再想想 ]      │
└──────────────────────────────────┘
```

**规则：**
- 单个文件删除：二次确认（无需输入文件名）
- 批量删除（>3 个文件）：二次确认 + 输入文件数量
- 删除文件夹：二次确认 + 输入文件夹名
- 所有删除操作均移至回收站，**不执行永久删除**

## 5. 权限与安全

### 5.1 允许的操作范围

默认仅允许操作以下位置：
- 桌面（Desktop）
- 文档（Documents）
- 下载（Downloads）
- 用户自定义的白名单目录

### 5.2 禁止的操作

- 系统目录（`C:\Windows`、`C:\Program Files`）
- 其他用户目录
- 注册表
- 系统进程

### 5.3 越权访问提示文案

当用户请求操作不在白名单内的路径时：

> **治愈系：** "这个地方我不太敢乱动呢...是系统重要文件哦，弄坏了就糟糕了~ 要不我帮你看看桌面和文档里的？"
>
> **搞笑系：** "哎哟！这可是系统禁区，我要是乱碰被抓了可就要进小黑屋了！换个地方吧~"
>
> **高冷系：** "该路径不在操作范围内。出于安全考虑，我只能访问你的个人文件夹。"
>
> **助手系：** "为了系统安全，我目前只能操作桌面、文档和下载文件夹。如果需要其他位置，可以在设置里添加白名单目录。"

## 7. 边界场景处理

### 7.1 文件已存在（复制/移动目标冲突）

**表现：** 弹出选项卡片

```
┌───────────────────────────────────┐
│  ⚠️ 目标位置已存在同名文件         │
│                                   │
│  已有：report.pdf (2.1 MB, 昨天)  │
│  新的：report.pdf (3.0 MB, 今天)  │
│                                   │
│  [ 覆盖 ] [ 保留两个 ] [ 取消 ]   │
└───────────────────────────────────┘
```

- "保留两个"自动重命名为 `report (1).pdf`

### 7.2 路径不存在

**处理：** 自动提示是否创建

> 宠物："这个路径还不存在呢，要我帮你先创建文件夹吗？"
> [ 创建并继续 ] [ 取消 ]

### 7.3 文件太大（> 100MB 复制/移动）

**处理：** 提示预计时间，并显示进度条

> 宠物："这个文件有 1.2 GB 呢，复制可能要花点时间，要继续吗？"
> [ 继续 ] [ 取消 ]

执行中显示进度：`正在复制... 45% (540 MB / 1.2 GB)`

### 7.4 文件被占用

**处理：** 友好提示 + 建议

> 宠物："这个文件好像正在被其他程序使用呢，要不先关掉再试？"
> [ 重试 ] [ 算了 ]

### 7.5 批量操作数量过多（> 50 个文件）

**处理：** 先展示操作预览

> 宠物："检测到 128 个文件需要处理，我先列一下：
> - 图片：45 个 → 图片文件夹
> - 文档：38 个 → 文档文件夹
> - 其他：45 个 → 其他文件夹
>
> 确定要整理吗？"
> [ 开始整理 ] [ 取消 ]

### 7.6 搜索无结果

> 宠物（治愈系）："没有找到呢...要不换个关键词试试？"
> 宠物（搞笑系）："找遍了都没有！这文件怕不是长腿跑了？"
> 宠物（高冷系）："未找到匹配文件。请确认关键词。"
> 宠物（助手系）："没有找到匹配的文件。建议：1. 检查文件名拼写 2. 扩大搜索范围 3. 尝试模糊搜索"

### 7.7 网络路径/外部存储不可用

> 宠物："这个位置现在连接不上呢，可能是 U 盘拔了或者网络断了？"

### 7.8 操作中途取消

- 文件操作支持取消（利用 Python 的信号中断）
- 已复制的部分文件会保留，并提示用户
- 宠物："已经停下来了。不过有几个文件已经复制过去了，要清理掉吗？"

## 8. 技术接口约定（给开发参考）

### 8.1 工具调用协议

AI 通过结构化 JSON 描述工具调用意图，前端解析后展示确认卡片：

```json
{
  "tool": "file_copy",
  "params": {
    "source": "C:\\Users\\xxx\\Desktop\\简历.pdf",
    "destination": "D:\\备份\\"
  },
  "risk_level": "medium",
  "description": "复制 简历.pdf 到 D:\\备份\\"
}
```

### 8.2 风险等级与确认策略

| 风险等级 | 确认方式 | 示例操作 |
|----------|----------|----------|
| `low` | 无需确认，直接执行 | 搜索、查看信息、打开文件 |
| `medium` | 单次确认 | 复制、移动、重命名、创建文件夹 |
| `high` | 二次确认 | 删除文件/文件夹 |

### 8.3 Python 工具脚本规范

- 位置：`tools/` 目录
- 每个工具一个 Python 文件
- 统一输入/输出 JSON 格式
- 通过 `child_process.execFile` 从 Electron 主进程调用
- 超时限制：单文件操作 30 秒，批量操作 5 分钟

- **白名单目录管理：** 可添加/删除允许操作的目录

- **Python 路径配置：** 自动检测或手动指定 Python 解释器
