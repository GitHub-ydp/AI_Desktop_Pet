## 3. 场景测试

### 3.1 反复提及场景

**步骤**：
1. 创建记忆 M，初始 stability=168h
2. 第 1 次强化 → 记录 S1
3. 第 2 次强化（间隔 >1 小时）→ 记录 S2
4. 第 3 次强化（间隔 >1 小时）→ 记录 S3

**验证**：
| 编号 | 检查点 | 期望 |
|------|-------|------|
| S-RP-01 | trigger_count | = 3 |
| S-RP-02 | S1 vs 初始 S | S1 > 168h，约 2 倍（~370h） |
| S-RP-03 | S3 vs S1 | S3 > 2 × S1（累积增长） |
| S-RP-04 | S3 约等于多少 | ~1200h（约 50 天半衰期） |

### 3.2 一次性重要事件保护

**步骤**：
1. 在 mood=10（极度悲伤）时创建记忆 "我妈妈去世了"
2. 等待 180 天（模拟时间推移）
3. 计算该记忆的 R 值

**验证**：
| 编号 | 检查点 | 期望 |
|------|-------|------|
| S-IE-01 | 初始 emotional_weight | = 1.4（mood=10, \|10-50\|/50=0.8, 1+0.5*0.8=1.4） |
| S-IE-02 | 初始 stability | = 168 × 1.4 = 235.2h（情感权重提升初始稳定性） |
| S-IE-03 | 180天后 R 值 | R = (1 + 19/81 * 4320/235.2)^(-0.5)，需大于 0.1 才不被软删除 |
| S-IE-04 | emotional_weight >= 1.3 | 即使 R < 0.01，该记忆免于硬删除 |
| S-IE-05 | 事实提取兜底 | 该事件应被 LLM 事实提取器提取为 event 类型的 memory_facts |

**S-IE-03 数值验算**：
- S = 235.2h, t = 4320h
- R = (1 + 0.2346 * 4320 / 235.2)^(-0.5)
- R = (1 + 4.308)^(-0.5)
- R = 5.308^(-0.5)
- R ≈ 0.434

结论：S=235.2h 时 180 天后 R ≈ 0.434 > 0.1，**不会被软删除**。

**补充场景（mood=5, 极端情况）**：
- emotional_weight = 1 + 0.5 * |5-50|/50 = 1 + 0.5*0.9 = 1.45
- 初始 S = 168 * 1.45 = 243.6h
- 180天后 R = (1 + 0.2346 * 4320/243.6)^(-0.5) ≈ 0.45 > 0.1

### 3.3 正常遗忘曲线

**步骤**：
1. 创建普通记忆 M（mood=50, EW=1.0, S=168h）
2. 不进行任何强化
3. 在不同时间点检查 R

**验证**：
| 编号 | 时间点 | 期望 R 值 | 状态 |
|------|-------|---------|------|
| S-NF-01 | 1 天后 | ≈ 0.875 | 正常 |
| S-NF-02 | 7 天后 | ≈ 0.618 | 开始模糊 |
| S-NF-03 | 30 天后 | ≈ 0.340 | 大部分遗忘 |
| S-NF-04 | 90 天后 | ≈ 0.196 | 几乎遗忘 |
| S-NF-05 | 120 天后 | < 0.15 | 接近休眠 |
| S-NF-06 | ~210 天后 | < 0.1 | 进入休眠（软删除） |

**S-NF-06 精确计算**：需要求解 (1 + 0.2346 * t/168)^(-0.5) = 0.1
- (1 + 0.2346t/168)^(-0.5) = 0.1
- 1 + 0.2346t/168 = 100
- 0.2346t/168 = 99
- t = 99 * 168 / 0.2346 ≈ 70,876h ≈ 2953 天 ≈ 8.1 年

**修正**：S=168h 的普通记忆在无强化情况下需要约 8 年才会 R < 0.1 进入休眠。这说明即使是普通记忆也有很长的保留期。30 天后 R≈0.34 仍远高于 0.1 阈值。

### 3.4 防抖机制验证

**步骤**：
1. 创建记忆 M
2. 连续调用 5 次 getContext（间隔 < 1 小时），命中 M
3. 等待 1 小时 1 分钟
4. 再调用 1 次 getContext

**验证**：
| 编号 | 检查点 | 期望 |
|------|-------|------|
| S-DB-01 | 步骤 2 后 trigger_count | = 1（只有第一次生效） |
| S-DB-02 | 步骤 2 后 stability | 只增长一次 |
| S-DB-03 | 步骤 4 后 trigger_count | = 2（冷却期过后第二次生效） |
| S-DB-04 | last_triggered_at 时间 | 步骤 2 后 = 第 1 次调用时间；步骤 4 后 = 最后调用时间 |

## 8. 验收标准总结

### 必须通过（P0）

- [ ] 所有单元测试通过（R 值精度 ±0.01，EW 精度 ±0.001）
- [ ] 搜索结果中 R < 0.1 的记忆不出现
- [ ] 高频记忆（多次强化）排名在搜索前 3
- [ ] 一次性重要事件（高情感权重）180 天后仍可回忆
- [ ] emotional_weight >= 1.3 的记忆不被硬删除
- [ ] 1 小时防抖正确工作
- [ ] v6 → v7 迁移成功，无数据丢失
- [ ] 不出现 better-sqlite3 同步错误或 SQLITE_BUSY

### 应当通过（P1）

- [ ] getContext 强化开销 < 5ms
- [ ] 旧数据兼容（NULL 字段降级处理）
- [ ] memory_facts personal 类型可保存
- [ ] 迁移幂等性
- [ ] 提醒/心情/事实提取等现有系统不受影响

### 建议通过（P2）

- [ ] 10K 记录搜索 < 100ms
- [ ] 饱和效应验证（高 S 增长变慢）
- [ ] 困难奖励验证（低 R 时强化增长更多）
- [ ] 迁移 1000 条记录 < 2 秒

## 附录 A：测试数据构造建议

```javascript
// 构造指定时间前的记忆
function createMemoryAt(hoursAgo, options = {}) {
  const now = Date.now();
  return {
    content: options.content || '测试记忆内容',
    created_at: now - hoursAgo * 3600000,
    updated_at: now - hoursAgo * 3600000,
    last_triggered_at: options.lastTriggered
      ? now - options.lastTriggered * 3600000
      : now - hoursAgo * 3600000,
    trigger_count: options.triggerCount || 0,
    stability: options.stability || 168.0,
    strength: options.strength || 1.0,
    emotional_weight: options.emotionalWeight || 1.0,
    mood: options.mood || 50
  };
}

// 示例：创建 90 天前的高情感记忆
const importantMemory = createMemoryAt(2160, {
  content: '我妈妈去世了',
  mood: 10,
  emotionalWeight: 1.4,
  stability: 235.2  // 168 * 1.4
});
```
